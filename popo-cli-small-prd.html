POPO CLI 小需求合集

3 个用户视角的能力补全

文档版本：V1.0 作者：青远(张振) 创建日期：2026-06-03 状态：待评审

背景

本期收集了 3 个 POPO CLI 用户反馈的小需求，聚焦在「让更多用户能用上 CLI」和「让 IM Skill 能力更完整」两个方向。

1需求清单

需求 1 · 账号扩展
支持 grp 账号使用 CLI

当前 POPO CLI 仅支持员工账号登录使用，部分场景需要使用 grp 账号（如团队公共账号、机器人账号等），希望能扩展支持。

需求范围

  • 支持使用 grp 账号完成 CLI 的首次登录 + 授权
  • 支持 grp 账号授权失效后收到授权卡片重新授权
  • 授权流程与员工账号保持一致，无特殊处理逻辑
需求 2 · IM 能力补全
IM Skill 支持给机器人发消息

当前 IM Skill 已支持给"人"发消息，但不支持给"机器人"发消息。希望补齐这个能力，让用户能通过 AI 一句话给机器人发消息。

需求范围

  • 给机器人发消息的逻辑与给人发消息保持一致：先搜索到机器人 → 再发消息
  • 搜索机器人时支持两种搜索方式：
    • 按机器人 名称 搜索
    • 按机器人 邮箱 搜索
  • 两种发送方式（给人、给机器人）的使用方法都写在同一个 Skill 文档里，让 AI 一次拿到完整调用方式
  • 减少 Skill 内部的多次工具调用：希望 AI 直接调用一次完成发送，不要在中间串联调用多个工具 / 方法

权限校验

  • 下达"给机器人发消息"指令的用户，必须在该机器人的「可使用范围」内
  • 不在使用范围内的用户尝试发送时，AI 应明确告知无权限，不执行发送

用户体验示例

用户指令

"帮我把这条消息发给 KM 小助手机器人，让它帮我搜一下相关知识"

期望结果

AI 调用 IM Skill 一次完成发送（搜索机器人 → 校验权限 → 发送消息），中间不需要串联调用多个工具 / 方法

需求 3 · IM 能力补全
IM Skill 支持查询群公告、群成员

当前 IM Skill 没有提供群相关的查询能力。用户在群运营、群分析等场景下需要查看群公告和群成员信息，希望能补上这两个能力。

需求范围

📢 群公告
  • ✅ 支持查询群公告内容
  • ❌ 不支持编辑群公告（本期不做）
👥 群成员
  • ✅ 支持查询群组中的所有成员列表
  • 每个成员需返回：姓名、邮箱、名片上的组织

查询字段示例

字段示例
姓名张振
邮箱zhangzhen19@corp.netease.com
名片上的组织效率工程部-云协作产品中心-AI工具组

用户体验示例

用户指令

"看一下「POPO CLI 互助交流群」里都有哪些人，按部门分组列一下"

期望结果

AI 调用 IM Skill 拉取群成员列表，按"组织"字段分组展示

2需求总览

#需求核心价值
1 支持 grp 账号使用 CLI 覆盖团队公共账号 / 机器人账号场景，扩大 CLI 用户覆盖面
2 IM Skill 支持给机器人发消息 统一发送入口，减少 AI 调用绕路，提升响应速度
3 IM Skill 支持查询群公告、群成员 补全群信息查询能力，支撑群运营 / 分析等场景
